# Dynamic dashboards API

Per-option reference for dynamic dashboards and their cards. For task-oriented documentation and worked examples, see the [Dynamic dashboards guide](./dynamic-dashboards.html).

Unlike most Avo configuration, these options are **not** set in `config/initializers/avo.rb` — they are columns and `settings` fields on the `avo_dashboards_dashboards` and `avo_dashboards_cards` records, edited through the ejected `DynamicDashboard`/`DynamicCard` Avo resources. The code blocks below show the value or snippet you enter in each field.

Every field whose value is Ruby or ERB is evaluated server-side with your app's full privileges. Read the [trust model](./dynamic-dashboards.html#the-trust-model) first.

## Dashboard options

<Option name="`name`">

The dashboard title shown to users and used to derive the slug when `slug` is blank.

```ruby
"Revenue overview"
```

- **Type:** String
- **Default:** `nil`
- **Validation:** required — the record won't save without it.

</Option>

<Option name="`slug`">

The dashboard's route param (`/avo/dashboards/<slug>`) and its id for collision detection. Auto-generated by parameterizing `name` when left blank.

```ruby
"revenue-overview"
```

- **Type:** String
- **Default:** derived from `name`
- **Validation:** required and unique — enforced by both a model validation and a database unique index. A slug that matches a code-defined dashboard's id makes this dashboard [shadowed](./dynamic-dashboards.html#when-a-dashboard-is-shadowed) (the code dashboard wins).

</Option>

<Option name="`description`">

Subtitle text shown under the dashboard title.

```ruby
"Weekly revenue and signups"
```

- **Type:** String
- **Default:** `nil`

</Option>

<Option name="`grid_cols`">

Number of columns in the dashboard's card grid.

```ruby
4
```

- **Type:** Integer
- **Default:** `3`
- **Values:** `3`, `4`, `5`, or `6`

</Option>

<Option name="`global_ranges`">

An optional list or hash of ranges shared by the dashboard's cards, mirroring a code dashboard's global ranges.

```ruby
{ "7 days": 7, "30 days": 30 }
```

- **Type:** Array or Hash (stored in the jsonb `settings`)
- **Default:** `[]`

</Option>

<Option name="`visible`">

A Ruby snippet deciding whether the dashboard appears in navigation. Stored in the `visible_snippet` field. Must return a boolean.

```ruby
current_user.admin?
```

- **Type:** Ruby snippet (String)
- **Default:** blank — the dashboard is visible
- **Context:** `dashboard`, `params`, `current_user` in scope
- **Fail mode:** fails **closed** — a raising snippet hides the dashboard and is logged
- **Cost:** runs on **every sidebar render**; runs only after the host `DynamicDashboard` policy authorizes the user. Keep it cheap.

</Option>

<Option name="`authorize`">

A Ruby snippet deciding whether the dashboard can be viewed. Stored in the `authorize_snippet` field. Must return a boolean.

```ruby
current_user.can?(:view_revenue)
```

- **Type:** Ruby snippet (String)
- **Default:** blank — authorized
- **Context:** `dashboard`, `params`, `current_user` in scope
- **Fail mode:** fails **closed** — a raising snippet denies access and is logged. Runs only after the host `DynamicDashboard` policy authorizes the user.

</Option>

## Card options

These apply to every dynamic card regardless of type.

<Option name="`card_type`">

Which kind of card this is. Drives which type-specific options apply; switching it discards the options that belong to the old type.

```ruby
"metric"
```

| Value       | Behavior                                              |
| ----------- | ----------------------------------------------------- |
| `metric`    | A single big number from a `query` snippet            |
| `chartkick` | A chart from a `query` snippet + `chart_type`         |
| `partial`   | Custom markup from a `partial` path or inline `erb_body` |
| `divider`   | A labelled separator between cards                     |

- **Type:** enum (String)
- **Default:** `nil`
- **Validation:** required
- **Editable:** yes — changing it strips options belonging to the previous type on save.
- **Fail mode:** an unknown or corrupt `card_type` (or a `code_class` that no longer resolves) renders the [error card](./dynamic-dashboards.html#error-states) for that card only — its siblings and the dashboard render normally.

</Option>

<Option name="`source`">

Where the card's behavior comes from.

```ruby
"query"
```

| Value        | Behavior                                                              |
| ------------ | -------------------------------------------------------------------- |
| `query`      | You write a Ruby `query` snippet                                      |
| `code_class` | You [reuse a registered code card class](./dynamic-dashboards.html#reuse-a-code-defined-card) |

- **Type:** enum (String)
- **Default:** `"query"`
- **Editable:** **no** — fixed after create (switching sources would orphan the other behavior column).

</Option>

<Option name="`label`">

The card's title.

```ruby
"New users"
```

- **Type:** String
- **Default:** `nil`

</Option>

<Option name="`position`">

Sort order of the card within its dashboard.

```ruby
10
```

- **Type:** Integer
- **Default:** `nil`

</Option>

<Option name="`query`">

The Ruby snippet that produces the card's data — for `source: query` cards. Its **return value** is the result; you do not call `result(...)`.

```ruby
User.where(created_at: 1.week.ago..).count
```

- **Type:** Ruby snippet (String)
- **Default:** blank — the card renders an empty result (a metric shows `0`, a chart shows an empty chart), never an error
- **Context:** `card`, `dashboard`, `arguments`, `params`, `current_user` in scope
- **Fail mode:** a snippet that **raises** renders the [error card](./dynamic-dashboards.html#error-states) (isolated to the card's frame); a metric `nil` result renders as `0`; an unplottable chart shape renders the error state.

</Option>

<Option name="`code_class`">

The name of a code-defined card class to reuse — for `source: code_class` cards. Selectable only from the allowlist (`Avo::Cards::BaseCard` descendants minus the gem's internal `Dynamic*` classes); resolved through the allowlist at render time, **never** `constantize`d.

```ruby
"Avo::Cards::UsersCount"
```

- **Type:** String (a class name)
- **Default:** `nil`
- **Values:** any allowlisted code-card class name
- **Fail mode:** a name that no longer resolves renders the [error card](./dynamic-dashboards.html#error-states) naming the class and logs a warning.

:::warning
The reused card's original dashboard `authorize` proc does **not** travel with it. Gate the dynamic dashboard yourself.
:::

</Option>

## Metric card options

Apply when `card_type` is `metric`.

<Option name="`prefix`">

Text placed before the value.

```ruby
"$"
```

- **Type:** String
- **Default:** `nil`

</Option>

<Option name="`suffix`">

Text placed after the value.

```ruby
"%"
```

- **Type:** String
- **Default:** `nil`

</Option>

<Option name="`format`">

A Ruby snippet that formats the query result. Its return value replaces the default formatting.

```ruby
number_to_currency value
```

- **Type:** Ruby snippet (String)
- **Default:** blank — the card uses default formatting (`number_to_social value.to_i`)
- **Context:** everything the `query` context has, **plus** `value` (the query result) and Rails' number helpers (e.g. `number_to_currency`, `number_to_social`)
- **Fail mode:** falls back to the **default formatting** (logged), *not* an error card — a formatting bug never hides the number.

</Option>

## Chartkick card options

Apply when `card_type` is `chartkick`. Requires the `chartkick` gem.

<Option name="`chart_type`">

The Chartkick chart to render.

```ruby
"area_chart"
```

- **Type:** String
- **Default:** `nil`
- **Values:** `line_chart`, `pie_chart`, `column_chart`, `bar_chart`, `area_chart`, `scatter_chart`

</Option>

<Option name="`chart_options`">

Options forwarded to Chartkick, entered as JSON.

```json
{ "library": { "plugins": { "legend": { "display": true } } } }
```

- **Type:** JSON (String) — parsed and deep-symbolized before it reaches Chartkick
- **Default:** blank
- **Validation:** must be valid JSON — an unparseable value fails to save with a clear error.

</Option>

<Option name="`flush`">

Whether to offset the chart's default padding so it sits flush in the card. Set it `false` to customize `scale`/`legend`.

```ruby
true
```

- **Type:** Boolean
- **Default:** the card base default (`true`)

</Option>

<Option name="`legend`">

Show or hide the chart legend (takes effect once `flush` is `false`).

```ruby
false
```

- **Type:** Boolean
- **Default:** the card base default

</Option>

<Option name="`scale`">

Show or hide the chart scale (takes effect once `flush` is `false`).

```ruby
false
```

- **Type:** Boolean
- **Default:** the card base default

</Option>

## Partial card options

Apply when `card_type` is `partial`. Provide **one** of the two.

<Option name="`partial`">

Path to a partial to render.

```ruby
"avo/cards/my_card"
```

- **Type:** String (partial path)
- **Default:** `nil`

</Option>

<Option name="`erb_body`">

An inline ERB string rendered in place, with standard Rails auto-escaping.

```erb
<div class="p-4"><%= dashboard.name %></div>
```

- **Type:** ERB snippet (String)
- **Default:** `nil`
- **Locals:** `card`, `dashboard`, `arguments`, `value`, `params`, `current_user`
- **Escaping:** `<%= %>` output is HTML-escaped by default
- **Fail mode:** renders the [error card](./dynamic-dashboards.html#error-states) on failure

:::danger
`raw(...)` / `html_safe` on `params`, request, or user-sourced data is a stored-XSS vector against every viewer. See the [trust model](./dynamic-dashboards.html#the-trust-model).
:::

</Option>

## Shared card options

Apply to every card type (never discarded when the type changes).

<Option name="`cols`">

Card width in grid columns.

```ruby
2
```

- **Type:** Integer
- **Default:** the card base default (`1`)
- **Values:** `1`–`6`

</Option>

<Option name="`rows`">

Card height in grid rows.

```ruby
2
```

- **Type:** Integer
- **Default:** the card base default (`1`)
- **Values:** `1`–`6`

</Option>

<Option name="`ranges`">

The range options offered in the card's header dropdown, mirroring a code card's `ranges`.

```ruby
{ "7 days": 7, "30 days": 30, All: "ALL" }
```

- **Type:** Array or Hash (stored in jsonb `settings`)
- **Default:** blank

</Option>

<Option name="`refresh_every`">

Auto-refresh interval, in seconds. The card re-fetches its frame on this interval.

```ruby
600
```

- **Type:** Integer (seconds)
- **Default:** `nil` (no auto-refresh)
- **Cost:** polling re-runs the card's `query` each cycle with no per-card timeout — see the [trust model](./dynamic-dashboards.html#the-trust-model). Ignored for [previews](./dynamic-dashboards.html#preview-before-saving).

</Option>

<Option name="`display_header`">

Whether to show the card's title and ranges header. Hide it to fill the whole card (e.g. an embedded map).

```ruby
false
```

- **Type:** Boolean
- **Default:** the card base default (`true`)

</Option>

<Option name="`discreet_description`">

A small info icon with a tooltip in the card's corner, for secondary context.

```ruby
"Counts only confirmed users."
```

- **Type:** String
- **Default:** `nil`

</Option>

<Option name="`invisible`">

Hides the card (for a `divider`, renders it with no line/label — spacing only). A card hidden this way also 404s on a direct frame request, so it isn't a pseudo-authorization backdoor.

```ruby
true
```

- **Type:** Boolean
- **Default:** `false`

</Option>

<Option name="`arguments`">

Arbitrary values passed to the card, entered as JSON. Read in a `query` snippet or a reused code card via `arguments[:key]`.

```json
{ "scope": "active", "threshold": 100 }
```

- **Type:** JSON (String) — deep-symbolized before it reaches the card
- **Default:** `{}`
- **Values:** JSON-expressible only — strings, numbers, booleans, arrays, hashes. No Ruby objects, procs, or symbols-as-values.

</Option>

## Snippet context

What each evaluated surface receives. Every surface is evaluated through [`Avo::ExecutionContext`](./execution-context), so its defaults (`params`, `current_user`, `context`, `view_context`) are always available; the columns below list what each surface adds on top.

| Surface     | Where            | Adds to scope                                   | Returns        | Fail mode           |
| ----------- | ---------------- | ----------------------------------------------- | -------------- | ------------------- |
| `query`     | metric / chart card | `card`, `dashboard`, `arguments`             | the data value | error card          |
| `format`    | metric card      | `card`, `dashboard`, `arguments`, `value`, number helpers | formatted value | default formatting |
| `erb_body`  | partial card     | `card`, `dashboard`, `arguments`, `value` (as locals) | HTML     | error card          |
| `visible`   | dashboard        | `dashboard`                                     | boolean        | closed (hidden)     |
| `authorize` | dashboard        | `dashboard`                                     | boolean        | closed (denied)     |

- Blank snippets short-circuit to the surface default without evaluating — `nil` for `query`/`format`, `true` for `visible`/`authorize`.
- `visible`/`authorize` results are memoized per request (keyed by record + surface).
- A malformed snippet (`SyntaxError`) is caught like any other failure — it never escapes to 500 a page.
